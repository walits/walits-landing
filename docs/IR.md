### MPC Technology (CGGMP21)

**Why CGGMP21?**
- **Industry Standard**: Fireblocks, Coinbase 등 글로벌 기업 채택
- **Threshold Signature**: 2-of-3, 3-of-5 등 유연한 정책
- **No Single Point of Failure**: 키가 한 곳에 모이지 않음
- **Proactive Security**: Key Refresh로 장기 안전성 확보

**Implementation Details**
```
Key Generation (DKG)
├─ Round 1: Commitment
├─ Round 2: ZK Proof Exchange
├─ Round 3: Share Distribution
└─ Output: Private Key Shares (never reconstructed)

Transaction Signing (TSS)
├─ Input: Transaction Hash
├─ Round 1: Nonce Commitment
├─ Round 2: Nonce Share
├─ Round 3: Signature Share
└─ Output: Full Signature (ECDSA/EdDSA)
```

**Supported Curves**
- **ECDSA**: secp256k1 (Bitcoin, Ethereum, Tron)
- **EdDSA**: ed25519 (Solana, XRP)

**Security Guarantees**
- **Adversary Threshold**: t < n/2 (2-of-3에서 1개 노드 compromise 가능)
- **Abort Security**: 악의적 노드 감지 시 프로토콜 중단
- **Simulation-based Proof**: 학술적으로 증명된 보안성

### HSM Integration

**Why HSM?**
- **Physical Security**: 물리적 해킹 방지
- **FIPS 140-2 Level 3**: 금융기관 표준
- **Tamper-Resistant**: 침입 시 자동 삭제

**Supported HSMs**
- **Thales Luna**: 글로벌 표준
- **AWS CloudHSM**: 클라우드 네이티브
- **nCipher**: 금융권 선호

### TEE (Trusted Execution Environment)

**Why TEE?**
- **Memory Encryption**: 런타임 중 메모리 암호화
- **Attestation**: 코드 무결성 증명
- **Performance**: HSM 대비 100x 빠름

**Supported TEEs**
- **Intel SGX**: 현재 사용 중
- **Intel TDX**: 차세대 (2026 로드맵)
- **AMD SEV**: 멀티 클라우드 대비

### B. CGGMP21 Protocol Flow

**Key Generation (DKG)**
```
Party 1              Party 2              Party 3
   │                    │                    │
   ├─ Round 1: Generate Random Commitment ──┤
   │◄───────── Broadcast Commitment ────────┤
   │                    │                    │
   ├─ Round 2: Generate ZK Proof ───────────┤
   │◄───────── Exchange ZK Proofs ──────────┤
   │                    │                    │
   ├─ Round 3: Distribute Key Shares ───────┤
   │◄───────── Receive Shares ──────────────┤
   │                    │                    │
   ▼                    ▼                    ▼
Private Key Share 1  Share 2            Share 3
(Never Reconstructed)
```

**Transaction Signing (TSS)**
```
Party 1              Party 2              Party 3
   │                    │                    │
   ├─ Input: TX Hash ──┼────────────────────┤
   │                    │                    │
   ├─ Round 1: Nonce Commitment ────────────┤
   │◄───────── Broadcast Commitment ────────┤
   │                    │                    │
   ├─ Round 2: Nonce Share ─────────────────┤
   │◄───────── Exchange Shares ─────────────┤
   │                    │                    │
   ├─ Round 3: Signature Share ─────────────┤
   │◄───────── Combine Shares ──────────────┤
   │                    │                    │
   ▼                    ▼                    ▼
      Full Signature (ECDSA/EdDSA)
      (Valid for Blockchain)
```

    aum_fee = {

    }

        2026: 3000, 2027: 500000, 2028: 5000000,

    }

    avg_txn_size = 50000  # KRW
